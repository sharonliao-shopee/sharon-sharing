## Page 1-1｜Sharon 怎么使用 Codex：入门

### 页面标题

入门：从基础问答到输出本地文件

### 页面说明

任务边界清楚、结果容易判断。先通过对话解决问题，再让 Codex 直接交付可以继续编辑和使用的文件。

### 1.1｜基础问答

#### 我怎么做的

直接提出需要理解、比较或整理的问题，通过多轮对话补充背景并收敛答案。

#### 实际例子

- **理解和比较 AI 工具：**让 Codex 对比 Claude、Claude Code 与 OpenClaw 的定位和适用场景，快速建立基础认知。
- **快速了解新话题：**让 Codex 搜索并整理 LLM、Agent 等领域资讯，把零散信息归纳成容易理解的结论。

#### 用了 Codex 的什么能力 / 功能

- 理解问题并组织答案
- 搜索、比较和归纳信息

#### 核心价值

快速补齐背景知识，形成初步判断。

### 1.2｜输出本地文件

#### 我怎么做的

说明内容、格式和保存位置，让 Codex 将结果直接生成并保存为本地文件。

#### 实际例子

- **生成工作材料：**把年中自评、团队奖和提名材料整理成可继续编辑的 Markdown 或 Word 文件。
- **整理和转换文件：**将零散资料输出为结构化文档、表格或其他需要的文件格式，并直接保存到本地工作区。

#### 用了 Codex 的什么能力 / 功能

- 生成 Markdown、Word、Excel 等本地文件
- 按指定格式保存、修改和转换文件


#### 核心价值

从“得到一个答案”推进到“拿到一个可以直接使用的文件”。

### 本页结论

入门阶段，我开始从“问 Codex”走向“让 Codex 交付一个具体结果”。

## Page 1-2｜Sharon 怎么使用 Codex：基础

### 页面标题

基础：连接真实知识与多个系统，完成闭环工作

### 页面说明

Codex 开始进入真实工作环境：读取复杂知识、连接不同系统，并将信息转成可以持续跟进的行动。

### 1.3｜连接 Google Workspace，完成复杂知识任务

#### 我怎么做的

让 Codex 读取 Google Docs、Sheets、Slides 等资料，跨文件查找事实、梳理关系并完成任务。

#### 实际例子

- **整理 Diana Topic Setup 规范：**从分散的项目资料中提取设置要求、操作步骤和注意事项，整理成团队可使用的统一说明。
- **搭建知识管理架构：**读取多个文件和目录，归纳知识分类、关系与缺口，形成清晰的知识库结构。

#### 用了 Codex 的什么能力 / 功能

- 读取和检索 Google Workspace 中的多份资料
- 综合复杂知识并输出结构化结果

#### 核心价值

减少人工查找和拼接资料的时间，让复杂知识直接服务于具体任务。

### 1.4｜接入不同系统，完成闭环工作

#### 我怎么做的

让 Codex 从消息、日历和工作文档中识别事项，生成待办、安排跟进，并通过定时任务持续检查。

#### 实际例子

- **从沟通信息生成行动清单：**读取 SeaTalk 和邮件中的重要消息，识别待回复事项、负责人和时间点，再结合 Google Workspace 材料生成待办。
- **持续跟进行程与任务：**读取日历中的会议和行程，整理会前准备与后续行动，并设置定时任务按计划提醒或检查进展。

#### 用了 Codex 的什么能力 / 功能

- 跨 SeaTalk、邮件、日历和 Google Workspace 读取并匹配信息
- 生成待办、安排跟进并执行定时任务

#### 核心价值

把“读取信息—识别行动—持续跟进”连接成完整工作闭环。

### 本页结论

Codex 开始从资料助手升级为可以跨系统推进工作的协作伙伴。

进阶 + 高阶：从低代码开发到可复用、可迁移的 Skill

#### 我怎么做的

描述目标、交互和视觉要求，让 Codex 编写 HTML、修改代码、连接 GitHub，并根据反馈持续迭代。

- 使用 GitHub 管理、更新和验证代码

#### 核心价值

用较低门槛把想法快速变成可以展示、试用和继续迭代的产品原型。

### 1.6｜编写可复用、可迁移的 Skill

#### 我怎么做的

从多次同类任务中提炼稳定的输入要求、执行步骤、判断标准和输出格式，再封装成 Skill。

#### 用了 Codex 的什么能力 / 功能

- 将个人方法封装为 Skill
- 打包参考资料、脚本和测试，使能力可迁移并持续迭代

#### 核心价值

让一次成功的做法变成可重复调用、可带到其他环境继续使用的能力。

## Page 2｜Take away from AI transformation

### 页面标题

AI transformation 的关键，是让 AI 进入真实任务并持续留下方法

### 页面说明

我的使用路径从具体工作开始，逐步连接上下文、工具、成品与可复用能力。

### Takeaway 1｜从具体工作开始

写材料、读文档、做分析。真实任务越多，适合交给 AI 的步骤越清楚。

### Takeaway 2｜上下文决定上限

连接真实文档、数据、聊天记录和本地环境后，答案才开始贴近工作实际。

### Takeaway 3｜快速产出 Demo

研究、方案、PPT 和原型都可以先快速出一版，再与 Codex 边做边改。

### Takeaway 4｜经验变成数字资产

规则、流程、分析方法和表达偏好，可以被整理、测试、更新和复用。

### Takeaway 5｜人聚焦判断与责任

Codex 负责准备与执行；关键数字、政策、外部表达和最终决定由人确认。

### 演进关系

真实任务 Task → 连接资料与工具 Workflow → 沉淀 Skill 与知识 Capability

### 本页结论

先把一件工作做通，再决定哪些步骤值得复用、连接和自动化。

## Page 3-2｜从简单的小任务开始

### 页面标题

从简单的小任务开始：选择做完马上能用的工作

### 页面说明

第一次体验的关键不在于任务有多复杂，而在于资料真实、结果清楚、你能快速判断有没有帮助。

### 任务 A｜一页项目 Brief

#### 适合解决什么

快速看清项目背景、进展、风险和需要决定的事项。

#### 示例指令

> 请阅读 @项目材料，整理：  
> 1. 要解决的问题  
> 2. 当前进展与决定  
> 3. 风险、依赖与缺口  
> 4. 两周内关键行动  
> 5. 需要我决定的事项  
> 数字和结论必须有原始依据。

### 任务 B｜方案比较

#### 适合解决什么

用统一维度比较候选方案，并明确建议依赖的前提。

#### 示例指令

> 请阅读 @项目A 和 @项目B 的材料，先提出统一的比较维度，再逐项比较：  
> - 收益与成本  
> - 实施难度  
> - 风险与适用前提  
> 最后说明建议、假设、信息缺口，以及试点要验证什么。

### 任务 C｜会议行动清单

#### 适合解决什么

把会议记录转成决定、待办、PIC 和时间点。

#### 示例指令

> 请阅读 @会议转录，提取：  
> - 已确认的决定  
> - 未解决的分歧  
> - Action / PIC / Due date  
> - 待补数据和材料  
> - 下次会议前事项  
> 没有明确 PIC 或日期时标记“待确认”。

### 本页结论

选择标准：这周本来就要做 + 有真实资料 + 结果容易判断 + 做完可以立刻使用。

## Page 3-3｜给出清晰的指令

### 页面标题

给出清晰的指令：说明上下文、交付标准与边界

### 页面说明

像给一位新同事布置任务：背景越清楚，越容易得到真正能用的结果。

### 一条清晰指令的 6 个部分

1. 背景：为什么会发生
2. 目标：要解决什么
3. 受众：谁看、谁决定
4. 资料：读取哪些来源
5. 要求：篇幅、结构、格式
6. 边界：不能猜什么、哪些动作需要确认

### 完整示例｜给领导的项目汇报

> **背景：**我们准备向领导提议启动一个新项目，目前材料比较分散。  
> **目标：**让领导在 10 分钟内理解为什么要做、方案是什么，以及需要他支持什么。  
> **受众：**负责决策和资源支持的领导。  
> **资料：**读取项目 Brief、调研记录和现有数据。  
> **要求：**先给一份 5 页以内的 PPT 故事线，每页使用结论式标题；确认后再制作完整内容。  
> **边界：**不要虚构数据；无法确认的信息请标出；修改原文件或对外发送前先问我。  
> 完成后，请从领导视角检查：哪些结论证据不足，哪些问题最可能被追问。

### 本页结论

给出清晰的指令，但不需要一次追求完美。

## Page 3-4｜开始沉淀自己的方法

### 页面标题

开始沉淀自己的方法：从反复要求走向可复用 Skill

### 页面说明

用过三五次以后，把稳定的输入要求、步骤、判断标准、输出格式和人工确认边界留下来。

### 沉淀路径

真实任务 → 重复偏好 → 模板 → Skill

1. **识别真实任务：**先完成几次同类工作。
2. **提炼重复偏好：**找出自己反复强调的要求。
3. **整理模板或 SOP：**固定前两步形成的输入要求、执行步骤、参考资料和输出格式。
4. **创建 Skill：**将方法打包，并在实际使用中持续运行和迭代。

### 示例指令｜把经验提炼成模板

> 请回顾我们最近几次完成的同类任务，提炼我反复强调的：  
> 1. 输入资料要求；  
> 2. 分析或执行步骤；  
> 3. 判断标准；  
> 4. 输出格式；  
> 5. 必须人工确认的边界。  
> 先整理成一份可复用模板，并标出仍需我确认的内容。

### Plugin 与 Skill 的区别

- Plugin：Codex 可以访问哪些系统和资料
- Skill：Codex 应该按照什么方法完成这类工作

### 本页结论

从小任务开始，经过真实使用，再把验证有效的方法留下来。
